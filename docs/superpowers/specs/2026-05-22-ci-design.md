# GitHub Actions CI Design

## Context

`rust-analyzer-mcp` is a Rust 2024 stdio MCP server. The README already lists the local development gates:

- `cargo fmt --check`
- `cargo clippy --all-targets --all-features -- -D warnings`
- `cargo test --all`

The test suite includes protocol framing tests, workspace path safety tests, MCP stdio smoke tests, and a rust-analyzer smoke test that runs when `rust-analyzer` is available on `PATH`.

## Goals

- Add useful GitHub Actions coverage for pushes and pull requests to `main`.
- Keep CI small and readable.
- Catch both Linux and Windows behavior, especially path handling and stdio process behavior.
- Match the commands contributors already see in the README.
- Make dependency resolution reproducible by honoring the checked-in `Cargo.lock`.

## Workflow Design

Add `.github/workflows/ci.yml` with one workflow named `CI`.

The workflow triggers on:

- `push` to `main`
- `pull_request` targeting `main`

The workflow uses least-privilege token permissions:

- `contents: read`

The workflow uses these action references:

- `actions/checkout@v6`
- `dtolnay/rust-toolchain@stable`
- `Swatinem/rust-cache@v2`

The workflow contains two jobs.

### `quality`

Runs once on `ubuntu-latest`.

Steps:

- Check out the repository.
- Install stable Rust with `rustfmt` and `clippy`.
- Cache Cargo registry, git dependencies, and build output.
- Run `cargo fmt --check`.
- Run `cargo clippy --locked --all-targets --all-features -- -D warnings`.

### `test`

Runs on both `ubuntu-latest` and `windows-latest`.

The OS matrix uses `fail-fast: false` so a failure on one platform does not hide the other platform's result.

Steps:

- Check out the repository.
- Install stable Rust.
- Install the `rust-analyzer` component so the optional smoke test exercises the real tool in CI.
- Run `rust-analyzer --version` to prove the smoke test will not be skipped because the binary is missing.
- Cache Cargo registry, git dependencies, and build output.
- Run `cargo test --locked --all`.

## Non-Goals

- No release publishing, packaging, or binary uploads.
- No nightly toolchain testing.
- No scheduled CI.
- No MSRV matrix until the project explicitly chooses an MSRV.

## Verification

Before considering the workflow done, run the matching local checks:

- `cargo fmt --check`
- `cargo clippy --locked --all-targets --all-features -- -D warnings`
- `cargo test --locked --all`
